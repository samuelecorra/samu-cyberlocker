# **Intro Modulo 9**

Introduciamo con questo modulo una particolare estensione delle basi di dati tradizionali, in cui il sistema non si limita a **rispondere passivamente alle interrogazioni**, ma è in grado di **reagire automaticamente a determinati eventi**.

L’obiettivo del modulo è comprendere le **caratteristiche fondamentali delle basi di dati attive (Active Databases)**, cioè quei sistemi che incorporano **regole attive** capaci di monitorare, rilevare e gestire automaticamente condizioni specifiche.

Verranno analizzate:

- le **funzionalità dei DBMS** necessarie a supportare le regole attive;
    
- le **problematiche legate all’attivazione e all’interazione** tra più regole (priorità, conflitti, cicli);
    
- e infine alcune **applicazioni pratiche** in cui le basi di dati attive offrono vantaggi significativi, come il controllo automatico di vincoli, la gestione di eventi complessi o l’automazione di processi aziendali.

In sintesi, il modulo mostra come un database possa diventare **proattivo**, capace di **intervenire autonomamente** quando si verificano determinate situazioni, evolvendosi da semplice archivio di dati a **componente intelligente del sistema informativo**.
