## **Lezione 4: Shell scripting – Caratteristiche (I)**

### **1. Introduzione**

In questa lezione si approfondiscono le **caratteristiche della shell** di Linux e le funzionalità avanzate che ne fanno uno strumento potente non solo per l’amministrazione del sistema, ma anche per l’**analisi forense e la costruzione di script automatizzati**.  

---

### **2. Tipi di shell**

Esistono diversi tipi di shell nei sistemi UNIX-like. Le principali sono:

- **Bourne Shell (sh)** → la shell originaria di UNIX.
    
- **Korn Shell (ksh)** → evoluzione compatibile, con funzionalità aggiuntive.
    
- **C Shell (csh, tcsh)** → introduce sintassi simile al linguaggio C.
    
- **Bash (Bourne-Again Shell)** → la più diffusa in ambiente Linux e derivata da `sh`, oggi standard de-facto.
    

Bash (acronimo per Bourne Again SHell) è una shell sviluppata nell'ambito del progetto GNU come alternativa libera di Bourne shell. Il nome è un calembour* poiché Bourne again (un'altra Bourne shell) è omofono a born again (rinascita).

La scelta della shell è in parte **una questione di preferenza personale**, ma Bash rimane la più utilizzata per la sua **compatibilità, stabilità e potenza di scripting**.

*
**Calembour** è un termine originario della lingua francese che indica un particolare gioco di parole basato sull’omofonia di parole che si pronunciano in maniera identica o simile ma hanno significato diverso. Pare che la parola derivi dal conte Kalenberg di Vestfalia, l’ambasciatore tedesco presente alla corte di re Luigi XV di Francia, il quale pronunciava male le parole in francese e dava spesso luogo a ridicoli errori di comprensione da parte dei raffinati parigini, che presero a definire tutti i doppi sensi “kalenberg”, poi corrotto pian piano in “calembour”, man mano che si perse memoria di questo goffo ambasciatore straniero.

### **3. Selezionare e cambiare shell**

Ogni utente di un sistema UNIX possiede una **shell predefinita** associata al proprio account.  
Per visualizzarla si usa:

```bash
echo $SHELL
```

Per modificarla:

```bash
chsh [username]
```

Il sistema richiede quindi di specificare il **path completo della nuova shell**, ad esempio:

```bash
New shell [/bin/bash]: /bin/csh
```

Questa possibilità consente di **personalizzare l’ambiente di lavoro** e di adattarlo alle proprie esigenze operative o di analisi.

### **5. Comandi built-in e comandi esterni**

- **Comandi esterni:**  
    Sono programmi veri e propri presenti nel file system, solitamente in directory come `/bin`, `/usr/bin`, ecc.  
    Quando vengono richiamati, il sistema ne **carica l’eseguibile in memoria** e lo esegue.  
    Esempio:
    
    ```bash
    /bin/ls
    ```
    
- **Comandi interni (built-in):**  
    Sono riconosciuti ed eseguiti direttamente dalla shell, **senza caricare nuovi processi**.  
    Esempi:
    
    ```bash
    echo "testo"
    cd /home
    ```
    

Questa distinzione è importante anche in ambito forense, perché i comandi built-in **non lasciano tracce di processo** nel sistema, mentre quelli esterni sì.

### **6. Metacaratteri**

I **metacaratteri** sono simboli speciali che modificano il comportamento dei comandi o della shell stessa.  
Tra i più importanti:

| Metacarattere   | Funzione                                       |
| --------------- | ---------------------------------------------- |
| `>` `>>` `<`    | Redirezione dell’I/O (output/input)            |
| `               | `                                              |
| `*` `?` `[ ]`   | Wildcard per pattern di nomi file              |
| `` `comando` `` | Command substitution con backticks<br>Alt + 96 |
| `;`             | Esecuzione sequenziale                         |
| `               |                                                |
| `( … )`         | Raggruppamento di comandi                      |
| `&`             | Esecuzione in background                       |
| `"` `'`         | Quoting di stringhe e variabili                |
| `#`             | Commento                                       |
| `$`             | Espansione di variabile                        |
| `\`             | Carattere di escape                            |
| `<<`            | “Here document”: input inline per un comando   |

> I metacaratteri sono il linguaggio con cui la shell dialoga con l’utente e gli script con il sistema: comprenderli è come conoscere la grammatica del linguaggio operativo.

### **7. Redirezione dell’input e dell’output**

Ogni processo in esecuzione in Linux è associato a **tre flussi principali**:

|Flusso|Nome|Significato|
|---|---|---|
|**stdin (0)**|standard input|input, di solito da tastiera|
|**stdout (1)**|standard output|output, di solito sul terminale|
|**stderr (2)**|standard error|messaggi di errore|

La **redirezione** permette di cambiare la destinazione o la sorgente di questi flussi.

#### **Esempi**

```bash
ls > list.txt
```

→ salva l’output del comando `ls` nel file `list.txt`. Il file viene creato se non esiste.

```bash
ls >> list.txt
```

→ aggiunge l’output in coda al file esistente. Se non esiste, lo crea

```bash
mail utente@email.it < list.txt
```

→ usa il contenuto di `list.txt` come input per il comando `mail`.

```bash
rm file >& /dev/null
```

→ redireziona sia l’output che gli errori verso `/dev/null` (scarto).

> In forensics, la redirezione è utile per salvare log, documentare risultati e separare i flussi informativi senza alterare i dati originali.

### **8. Pipe**

Le **pipe** (o “catene di montaggio”) permettono di collegare l’**output di un processo** all’**input di un altro**, senza creare file intermedi.

#### **Esempio**

```bash
ls | wc -w
```

Il comando `ls` elenca i file, e il suo output viene inviato come input a `wc -w`, che conta le parole, restituendo il numero totale di file.  
Questo meccanismo consente di costruire **sequenze di elaborazione dati**, componendo strumenti semplici per ottenere operazioni complesse.

> Le pipe sono il cuore della filosofia UNIX: “_fai una cosa sola, ma falla bene_”.

---

### **9. Wildcards**

Le **wildcards** (caratteri jolly) vengono usate per indicare **pattern di nomi di file**.

| Simbolo | Significato                       | Esempio                                  |
| ------- | --------------------------------- | ---------------------------------------- |
| `*`     | Qualsiasi stringa (anche vuota)   | `*.c` → tutti i file con estensione `.c` |
| `?`     | Qualsiasi singolo carattere       | `prova0?.txt`                            |
| `[ ]`   | Qualsiasi carattere tra parentesi | `foto[0-9][0-9].jpg`                     |

> Le wildcards consentono di operare su **insiemi di file** in modo compatto, strumento prezioso per l’analisi massiva dei dati digitali.

### **10. Command substitution**

La **command substitution** permette di **inserire l’output di un comando** dentro un altro comando oppure di **assegnarlo a una variabile**.  
Serve quando vuoi che un comando utilizzi dinamicamente il risultato di un altro comando, come data, ora, conteggi, ricerche o qualsiasi output testuale.

#### **10.1 Sintassi storica (backtick) — DA NON USARE PIÙ**

```bash
`comando`
```

Questa è la forma originale, compatibile ma scomoda.  
Usa gli apici rovesciati (backtick), che:

- sono difficili da leggere;
    
- si confondono facilmente con l’apostrofo `'`;
    
- non permettono un annidamento leggibile (bisogna escapare i backtick interni con ```);
    
- creano confusione quando combinati con virgolette e pipe.
    

#### **Esempi (forma storica)**

```bash
echo "Data odierna: `date`"
echo "Utenti collegati: `who | wc -l`"
tar zcvf src-`date +%Y%m%d`.tgz src/
```

Funzionano, ma non sono consigliati.

---

#### **10.2 Sintassi moderna e raccomandata**

La forma attuale, standard POSIX e raccomandata ovunque, è:

```bash
$(comando)
```

Questa sintassi è:

- più leggibile
    
- facile da annidare
    
- chiara nelle pipe
    
- totalmente non ambigua
    
- universalmente raccomandata da ShellCheck e dalla documentazione Bash
    

#### **Esempi (forma moderna)**

```bash
echo "Data odierna: $(date)"
echo "Utenti collegati: $(who | wc -l)"
tar zcvf src-$(date +%Y%m%d).tgz src/
```

#### **Esempio di annidamento pulito**

```bash
echo "Oggi è: $(echo $(date +%A))"
```

Con i backtick, la stessa cosa diventerebbe quasi illeggibile:

```bash
echo "Oggi è: `echo \`date +%A\``"
```

---

#### **10.3 Uso nella creazione di variabili**

```bash
data=$(date)
utenti=$(who | wc -l)

echo "Oggi è: $data"
echo "Utenti collegati: $utenti"
```

---

#### **10.4 Regola pratica**

**Usa sempre `$(comando)` — evita i backtick.**

La sintassi moderna è la più robusta, leggibile e mantenibile, ed è quella che vedrai in tutti gli script professionali e nella documentazione aggiornata.

### **11. Sequenze condizionali e non condizionali**

#### **Sequenze non condizionali**

Il metacarattere `;` consente di eseguire comandi in sequenza:

```bash
date ; pwd ; ls
```

→ esegue i tre comandi uno dopo l’altro, indipendentemente dal risultato.

#### **Sequenze condizionali**

- `&&` → esegue il secondo comando **solo se il primo ha successo** (exit code = 0).
    
- `||` → esegue il secondo comando **solo se il primo fallisce** (exit code ≠ 0).
    

#### **Esempi**

```bash
gcc prog.c && ./a.out
gcc prog.c || echo "Compilazione fallita"
```

Questo approccio è molto utile negli script per controllare **flussi di esecuzione** in base agli esiti dei comandi precedenti.
