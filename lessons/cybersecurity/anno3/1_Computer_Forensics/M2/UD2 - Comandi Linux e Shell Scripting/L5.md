## **Lezione 5: Shell scripting – Caratteristiche (II)**

### **1. Introduzione**

Questa lezione conclude lo studio delle **caratteristiche della shell di Linux**, approfondendo gli aspetti pratici e logici che rendono la shell un vero linguaggio di programmazione:

- raggruppamento di comandi e processi in background,
    
- quoting e subshell,
    
- gestione delle variabili,
    
- costruzione di script eseguibili e variabili predefinite.
    

L’obiettivo è acquisire la piena padronanza del comportamento della shell in ambienti complessi, così da poter creare **strumenti automatizzati di analisi forense**.

---

### **2. Raggruppamento di comandi**

La shell consente di **raggruppare comandi** racchiudendoli tra parentesi tonde:

```bash
(date ; ls ; pwd) > out.txt
```

Questo costrutto:

- viene eseguito in una **subshell** separata (un processo figlio della shell principale);
    
- condivide con la shell principale i flussi standard (`stdin`, `stdout`, `stderr`);
    
- produce un **unico output redirezionabile**.
    

Esempio di confronto:

```bash
date ; ls ; pwd > out.txt
```

→ redireziona solo l’output di `pwd`.

```bash
(date ; ls ; pwd) > out.txt
```

→ redireziona **l’output dell’intero gruppo di comandi**.

> In ambito forense, il raggruppamento è utile per eseguire sequenze di acquisizione o filtraggio dati mantenendo la **coerenza logica del flusso di output**.

---

### **3. Esecuzione in background**

Un comando seguito dal metacarattere `&` viene eseguito in **background**, cioè in **parallelo** alla shell attiva:

```bash
find / -name passwd -print &
```

La shell:

- crea una **subshell** che esegue il comando;
    
- non cede il controllo della tastiera;
    
- consente di avviare altri processi contemporaneamente;
    
- mostra il **PID (Process ID)** del comando avviato.
    

Esempio di redirezione e background combinati:

```bash
find / -name passwd -print &> results.txt &
```

Il primo `&>` redireziona sia `stdout` che `stderr` nel file `results.txt`;  
il secondo `&` invia il processo in background.

> L’esecuzione in background è ideale per **operazioni lunghe**, come la scansione di interi dischi o la generazione di hash, che non richiedono interazione diretta.

### **4. Quoting**

Il **quoting** serve a controllare come la shell interpreta caratteri speciali, metacaratteri o variabili.

#### **Apice singolo `' '`**

Disattiva:

- wildcard (`*`, `?`),
    
- command substitution (`` ` ` ``),
    
- variable substitution (`$var`).
    

Esempio:

```bash
echo 3 * 4 = 12
echo '3 * 4 = 12'
```

→ nel primo caso `*` viene interpretato come wildcard, nel secondo viene trattato come carattere letterale.

#### **Doppio apice `"`**

Inibisce solo le wildcard, ma **mantiene attiva l’espansione delle variabili**:

```bash
echo "Il mio nome è $USER e oggi è `date`"
echo 'Il mio nome è $USER e oggi è `date`'
```

Nel primo caso la shell espande `$USER` e `date`, nel secondo li stampa letteralmente.

> In scripting forense, il quoting è fondamentale per **preservare stringhe e percorsi contenenti spazi, simboli o caratteri speciali**, evitando interpretazioni errate da parte della shell.

---

### **5. Subshell**

Ogni volta che si apre un terminale, viene lanciata una **shell principale**.  
Alcuni comandi o situazioni creano **subshell** (processi figlio):

- durante l’esecuzione di comandi raggruppati:
    
    ```bash
    (date ; ls ; pwd)
    ```
    
- quando viene eseguito uno **script di shell**;
    
- quando un processo viene mandato in **background**.
    

Le subshell ereditano l’ambiente della shell padre, ma possono avere **variabili locali** e **flussi I/O separati**.

---

### **6. Variabili nella shell**

Ogni shell gestisce due tipi principali di variabili:

#### **a. Variabili locali**

- Valide solo nella shell o nello script in cui vengono definite;
    
- Non ereditate dalle subshell;
    
- Utili per calcoli temporanei o parametri interni.
    

#### **b. Variabili di ambiente**

- Ereditate dalle shell figlie;
    
- Utilizzate per comunicazioni tra processi e per definire parametri globali del sistema.
    

Esempi di variabili d’ambiente predefinite:

```
$HOME, $PATH, $MAIL, $USER, $SHELL, $TERM
```

Per visualizzare tutte le variabili:

```bash
env
```

---

### **7. Creazione e utilizzo delle variabili**

#### **Assegnazione**

```bash
variabile=valore
variabile="valore con spazi"
```

#### **Accesso**

```bash
echo $variabile
echo ${variabile}
```

#### **Promozione a variabile d’ambiente**

```bash
export variabile
```

Le variabili dichiarate senza `export` restano locali; quelle esportate diventano accessibili anche nelle subshell.

---

### **8. Esempio pratico: variabili locali e d’ambiente**

```bash
firstname="Giovanni"
lastname="Rossi"
echo $firstname $lastname

export firstname
bash      # apre una nuova subshell
echo $firstname $lastname   # mostra solo firstname
```

La variabile `firstname` è stata esportata, mentre `lastname` no:  
solo la prima è visibile nella subshell.  
Chiudendo la subshell con `exit`, i valori della shell principale vengono ripristinati.

---

### **9. Script di shell**

Uno **script** è una sequenza di comandi salvata in un file di testo ed eseguita come programma.

#### **Creazione ed esecuzione**

1. Scrivere lo script in un file, es. `esempio.sh`.
    
2. Rendere il file eseguibile:
    
    ```bash
    chmod +x esempio.sh
    ```
    
3. Eseguire lo script:
    
    ```bash
    ./esempio.sh
    ```
    

Durante l’esecuzione:

- la shell determina **quale interprete** usare (tramite la prima riga `#!`);
    
- crea una **subshell**;
    
- passa il contenuto dello script come input alla shell.
    

---

### **10. Variabili built-in per lo scripting**

Durante l’esecuzione, la shell mette a disposizione alcune **variabili predefinite**:

|Variabile|Descrizione|
|---|---|
|`$$`|PID (Process ID) della shell corrente|
|`$0`|Nome dello script eseguito|
|`$1 ... $9`|Argomenti passati allo script|
|`${n}`|Alternativa alla sintassi `$n`|
|`$*`|Tutti gli argomenti come stringa unica|
|`$#`|Numero totale di argomenti passati|
|`$?`|Valore di uscita dell’ultimo comando|

Queste variabili consentono di creare **script parametrici**, capaci di adattarsi ai valori ricevuti da linea di comando.

---

### **11. Esercizi pratici**

#### **a. Uso di variabili locali e d’ambiente**

```bash
#!/bin/bash
a=${1}
echo $a

b=$a
echo $b

a=`echo Ciao!`
echo $a

a=`ls -l`
echo $a

exit 0
```

Questo script dimostra come una stessa variabile possa:

- ricevere input da argomento di linea di comando;
    
- essere riassegnata tramite command substitution;
    
- mostrare output multipli nella stessa sessione.
    

---

#### **b. Script “addizione”**

Obiettivo: scrivere uno script che **sommi due numeri** passati come argomenti.

**Esempi di esecuzione:**

```
addizione 1 5     → 6  
addizione 9 89    → 98  
addizione 5 ciao  → Errore: gli argomenti devono essere numerici
```

**Suggerimenti tecnici:**

- Utilizzare `let` per le operazioni aritmetiche;
    
- Accedere agli argomenti tramite `${1}`, `${2}`;
    
- Gestire gli errori con operatori condizionali `||`.
    

> Questo esercizio allena all’uso di **input, variabili, operatori logici e controllo degli errori**, elementi essenziali per costruire strumenti forensi automatizzati.
