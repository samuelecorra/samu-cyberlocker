## **Lezione 6: Shell scripting – Espressioni e strutture di controllo**

### **1. Introduzione**

Questa lezione segna il passaggio dal linguaggio descrittivo della shell alla sua **logica di programmazione vera e propria**.  
Attraverso lo studio delle **espressioni**, dei **test condizionali** e delle **strutture di controllo** (`if`, `case`, `while`, `until`, `for`), la shell diventa un **ambiente di scripting completo**, capace di elaborare decisioni, gestire cicli e reagire a condizioni logiche o numeriche.

In ambito **forense**, tali costrutti sono fondamentali per creare **script automatici** che eseguono analisi mirate: estrazione di log, scansione di directory sospette, verifica di timestamp o confronti tra hash di file.

---

### **2. Espressioni**

#### **Valutazione delle espressioni**

La shell, di base, **non supporta direttamente** la valutazione di espressioni aritmetiche o logiche.  
Per questo si utilizza l’utility **`expr`**, che:

- valuta un’espressione e invia il risultato su **stdout**;
    
- richiede che **ogni elemento sia separato da spazi**;
    
- obbliga a **proteggere i metacaratteri** (es. `*`) con un **backslash**;
    
- può produrre risultati **numerici o testuali**;
    
- può essere usata all’interno di una **command substitution** per assegnare il risultato a una variabile.
    

#### **Operatori principali**

- **Aritmetici:** `+ - * / %`
    
- **Confronto:** `= != > < >= <=`
    
- **Logici:** `& | !`
    
- **Stringhe:**
    
    - `match string regularExpression`
        
    - `substr string start length`
        
    - `length string`
        
    - `index string charList`
        

#### **Esempi pratici**

```bash
x=1
x=`expr $x + 1`
echo $x     # 2

echo `expr $x \* $x`   # 4
echo `expr length "cat"`   # 3
echo `expr index "donkey" "key"`  # 4
echo `expr substr "donkey" 4 3`   # key
```

> L’utility `expr` è utile negli script forensi per calcolare iterazioni, contatori di file, offset o posizioni di caratteri in stringhe complesse.

### **3. Exit status**

Quando un processo termina, ritorna un **exit status**, cioè un codice numerico che indica l’esito dell’esecuzione.

#### **Convenzioni UNIX**

- `0` → successo (**TRUE**)
    
- diverso da `0` → fallimento (**FALSE**)
    

#### **Strumenti correlati**

- `exit n` → forza la chiusura di uno script restituendo lo status `n`
    
- `$?` → variabile built-in che contiene l’exit status dell’ultimo comando
    

#### **Esempio**

```bash
#!/bin/bash
echo hello
echo $?        # 0 (successo)

gnègnègnè      # comando inesistente
echo $?        # diverso da 0 (errore)

exit 987       # chiusura esplicita
```

Output:

```
hello
0
./script.sh: line 5: gnègnègnè: command not found
127
```

> Il controllo dell’exit status è una **tecnica chiave di validazione**: consente di sapere se una procedura di copia, hashing o estrazione è stata completata con successo o ha generato un’anomalia.

### **4. Condizioni e test**

In Bash, le **condizioni logiche** sono strettamente legate all’exit status dei comandi.

#### **Esempio:**

```bash
if cmp a b > /dev/null
then
  echo "Files a and b are identical."
else
  echo "Files a and b differ."
fi
```

Se il comando `cmp` termina con exit status `0`, la condizione è **vera**; altrimenti è **falsa**.

### **5. Il comando `test`**

Il comando `test` valuta espressioni e restituisce `0` (vero) o `1` (falso).

#### **Operatori numerici**

|Operatore|Significato|
|---|---|
|`-eq`|uguale a|
|`-ne`|diverso da|
|`-gt`|maggiore di|
|`-ge`|maggiore o uguale|
|`-lt`|minore|
|`-le`|minore o uguale|

#### **Operatori booleani**

|Operatore|Significato|
|---|---|
|`! expr`|negazione|
|`expr1 -a expr2`|AND|
|`expr1 -o expr2`|OR|
|`\( \)`|raggruppamento|

#### **Confronti tra stringhe**

|Operatore|Significato|
|---|---|
|`str1 = str2`|uguali|
|`str1 != str2`|diversi|
|`-z string`|stringa vuota|
|`-n string`|stringa non vuota|

#### **Confronti su file**

|Operatore|Descrizione|
|---|---|
|`-d file`|esiste ed è directory|
|`-f file`|esiste ed è file regolare|
|`-r file`|leggibile|
|`-w file`|scrivibile|
|`-x file`|eseguibile|
|`-s file`|non vuoto|

### **6. Sintassi alternativa: `[ ]` e `[[ ]]`**

- `[ ]` → forma tradizionale, equivalente a `test`.
    
- `[[ ]]` → forma estesa (**keyword**), più sicura e versatile.
    

Vantaggi di `[[ ]]`:

- disattiva l’espansione dei nomi file (`*`, `?`);
    
- interpreta correttamente gli operatori logici `&&` e `||`;
    
- gestisce in modo più coerente le stringhe con spazi o caratteri speciali.
    

Esempio:

```bash
if [[ $x -gt 0 && $y -lt 10 ]]; then
  echo "Valori coerenti"
fi
```

### **7. Strutture di controllo condizionale**

#### **a. if – then – elif – else**

La struttura di base per il controllo condizionale.

```bash
if condizione1
then
  comandi1
elif condizione2
then
  comandi2
else
  comandi3
fi
```

> `elif` significa _else if_, e la sezione `else` è opzionale.  
> Negli script forensi viene spesso usata per distinguere percorsi di indagine (es. directory valide, immagini disco mancanti, errori di permessi).

---

#### **b. case – in – esac**

Alternativa a `if`, ideale per gestire **menu o scelte multiple**.

```bash
case expression in
  pattern1) comandi1 ;;
  pattern2) comandi2 ;;
  *)        comandi_default ;;
esac
```

Esempio pratico:

```bash
#!/bin/bash
case "$1" in
  "") lines=50 ;;
  *[!0-9]*) echo "Usage: `basename $0` usersnum"; exit 1 ;;
  *) lines=$1 ;;
esac
du -s /home/* | sort -gr | head -$lines
```

### **8. Strutture iterative**

#### **a. while – do – done**

```bash
while condizione
do
  comandi
done
```

Esegue i comandi finché la condizione è **vera**.  
Può essere interrotto con `break` o continuato con `continue`.

#### **b. until – do – done**

```bash
until condizione
do
  comandi
done
```

L’opposto di `while`: ripete il ciclo **fino a quando** la condizione diventa **vera**.

#### **c. for – in – do – done**

```bash
for variabile in lista
do
  comandi
done
```

Esegue una sequenza di comandi per ogni elemento della lista.

---

### **9. Esempi pratici di strutture di controllo**

#### **a. Esempio di `while`**

```bash
#!/bin/bash
# Tabelline
if [ "$1" = "" ]; then
  echo "Usage: $0 max"
  exit 1
fi
x=1
while [ $x -le $1 ]
do
  y=1
  while [ $y -le $1 ]
  do
    echo `expr $x \* $y`
    y=`expr $y + 1`
  done
  echo
  x=`expr $x + 1`
done
exit 0
```

#### **b. Esempio di `for`**

```bash
#!/bin/bash
for color in red yellow green blue
do
  echo one color is $color
done
```

Output:

```
one color is red
one color is yellow
one color is green
one color is blue
```

#### **c. Esempio di `case`**

```bash
#!/bin/bash
case "$1" in
  1) echo "administrator" ;;
  2) echo "student" ;;
  3) echo "teacher" ;;
  *) echo "Error" ;;
esac
```

#### **d. Script “bin-grep.sh”**

Ricerca stringhe specifiche all’interno di file binari:

```bash
#!/bin/bash
# bin-grep.sh: trova stringhe in file binari
EBADARGS=65
ENOFILE=66

if [ $# -ne 2 ]; then
  echo "Usage: `basename $0` string filename"
  exit $EBADARGS
fi

if [ ! -f "$2" ]; then
  echo "File \"$2\" does not exist."
  exit $ENOFILE
fi

for word in $(strings "$2" | grep "$1")
do
  echo $word
done
```

> Questo script è una versione “forense” di `grep`: estrae stringhe ASCII da file binari per verificare la presenza di frammenti significativi o prove testuali in aree non leggibili da editor tradizionali.

### **10. Conclusioni**

Con questa lezione si completa la panoramica sui **fondamenti del linguaggio shell**, che da semplice interprete di comandi diventa un **ambiente logico per l’analisi forense**.  
Abbiamo visto come:

- valutare **espressioni numeriche e testuali**;
    
- usare **test condizionali** e **exit status**;
    
- strutturare **cicli e condizioni** complesse;
    
- automatizzare la raccolta e l’elaborazione di dati digitali.
    

> In forensics, la shell scripting non è solo un linguaggio:  
> è lo **strumento del pensiero metodico** dell’analista,  
> capace di trasformare procedure complesse in sequenze controllate, tracciabili e ripetibili.
