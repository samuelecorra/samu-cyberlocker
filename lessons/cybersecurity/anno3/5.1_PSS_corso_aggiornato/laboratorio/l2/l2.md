## 1. Equivalenza tra formule logiche

Due formule proposizionali `P` e `Q` sono **equivalenti** — e scriveremo `P ≡ Q` — se possiedono la **stessa tavola di verità**, ovvero se per ogni possibile assegnazione di valori di verità alle variabili che vi compaiono, `P` e `Q` assumono lo stesso valore. [p. 1]

L'equivalenza logica è lo strumento fondamentale che ci consente di **riscrivere** una formula in una forma diversa ma con identico significato. In contesti di sicurezza del software, questo è cruciale: una specifica formale può essere semplificata o ristrutturata senza alterarne il contenuto semantico, rendendo il codice più leggibile, verificabile e manutenibile.

## 2. Le equivalenze fondamentali

Di seguito presentiamo sistematicamente tutte le leggi di equivalenza della logica proposizionale, raggruppate per categoria. [p. 2–7]

### 2.1 Idempotenza

L'applicazione ripetuta dello stesso connettivo con lo stesso operando non cambia il risultato:

$$P \lor P \equiv P$$
$$P \land P \equiv P$$

In notazione operativa:

```
P || P  ≡  P
P && P  ≡  P
```

Intuitivamente: dire _"piove o piove"_ equivale a dire _"piove"_; analogamente, _"piove e piove"_ è semplicemente _"piove"_.

### 2.2 Commutatività

L'ordine degli operandi non influenza il risultato per i connettivi simmetrici:

```
P || Q      ≡  Q || P
P && Q      ≡  Q && P
P <==> Q    ≡  Q <==> P
P <=!=> Q   ≡  Q <=!=> P
```

> [!WARNING]
> L'implicazione **non** è commutativa:
>
> ```
> P ==> Q  ≢  Q ==> P
> ```
>
> Questo è un errore frequentissimo. _"Se piove allora prendo l'ombrello"_ non equivale a _"Se prendo l'ombrello allora piove"_. In termini di sicurezza, invertire la direzione di un'implicazione in una policy di accesso può ribaltarne completamente il significato.

### 2.3 Associatività

Le parentesi possono essere spostate liberamente per i seguenti connettivi:

```
(P || Q) || R       ≡  P || (Q || R)
(P && Q) && R       ≡  P && (Q && R)
(P <==> Q) <==> R   ≡  P <==> (Q <==> R)
(P <=!=> Q) <=!=> R ≡  P <=!=> (Q <=!=> R)
```

Questo ci permette di scrivere catene di `||` o `&&` senza ambiguità riguardo al raggruppamento.

### 2.4 Contraddizione e terzo escluso

Due leggi fondamentali che coinvolgono una proposizione e la sua negazione:

```
!P && P   ≡  false      // Contraddizione
!P || P   ≡  true       // Terzo escluso (tertium non datur)
P ==> P   ≡  true       // Ogni proposizione implica sé stessa
```

La **contraddizione** afferma che è impossibile che una proposizione sia contemporaneamente vera e falsa. Il **terzo escluso** afferma che ogni proposizione è vera oppure falsa, senza vie di mezzo. Questi due principi sono i pilastri della logica classica bivalente.

### 2.5 Assorbimento

```
P || (P && Q)  ≡  P
P && (P || Q)  ≡  P
```

Se `P` è vera, allora `P || (P && Q)` è vera indipendentemente da `Q`; se `P` è falsa, sia `P` sia `P && Q` sono false, quindi il risultato è `P`. Ragionamento analogo per la seconda legge.

### 2.6 Distributività

I connettivi `||` e `&&` si distribuiscono l'uno attraverso l'altro, esattamente come la moltiplicazione si distribuisce sulla somma in aritmetica:

```
P || (Q && R)  ≡  (P || Q) && (P || R)
P && (Q || R)  ≡  (P && Q) || (P && R)
```

> [!IMPORTANT]
> La distributività è una delle leggi più utilizzate nella semplificazione di espressioni booleane e nella riscrittura di condizioni nei programmi. Ad esempio, un `if` con una condizione complessa può spesso essere ristrutturato in forma più leggibile applicando la distributività.

### 2.7 Leggi di De Morgan

Le leggi di De Morgan descrivono come la **negazione si distribuisce** attraverso congiunzione e disgiunzione, _scambiandole_:

```
!(P || Q)  ≡  !P && !Q
!(P && Q)  ≡  !P || !Q
```

In parole:

- La negazione di _"P o Q"_ equivale a _"non P **e** non Q"_.
- La negazione di _"P e Q"_ equivale a _"non P **o** non Q"_.

#### Dimostrazione della correttezza di De Morgan mediante tavola di verità

Verifichiamo `!(P || Q) ≡ !P && !Q`: [p. 8]

| P   | Q   | P \|\| Q | !(P \|\| Q) | !P  | !Q  | !P && !Q | !P \|\| !Q |
| --- | --- | -------- | ----------- | --- | --- | -------- | ---------- |
| F   | F   | F        | **V**       | V   | V   | **V**    | V          |
| F   | V   | V        | **F**       | V   | F   | **F**    | V          |
| V   | F   | V        | **F**       | F   | V   | **F**    | V          |
| V   | V   | V        | **F**       | F   | F   | **F**    | F          |

Le colonne `!(P || Q)` e `!P && !Q` sono identiche: l'equivalenza è verificata. ✓

Si noti anche che `!P || !Q` (ultima colonna) **non** coincide con `!(P || Q)`, ma coincide con `!(P && Q)` — è l'altra legge di De Morgan.

### 2.8 Doppia negazione

```
!!P  ≡  P
```

Negare due volte una proposizione la riporta al suo valore originale. Nei linguaggi di programmazione, `!!flag` è un idioma talvolta usato per forzare la conversione a booleano.

### 2.9 Equivalenze sull'implicazione

```
P ==> Q  ≡  !P || Q
P ==> Q  ≡  !Q ==> !P    // Contronominale
```

La prima equivalenza è fondamentale: l'implicazione può _sempre_ essere riscritta come disgiunzione di una negazione. La seconda è il **principio della contronominale**: per dimostrare che `P ==> Q`, possiamo equivalentemente dimostrare che `!Q ==> !P`.

### 2.10 Scomposizione della bicondizionale

```
P <==> Q  ≡  (P ==> Q) && (Q ==> P)
```

"P se e solo se Q" significa che `P` implica `Q` **e** `Q` implica `P`. Entrambe le direzioni devono valere contemporaneamente.

### 2.11 Semplificazioni con costanti

```
P || true    ≡  true       // Un OR con true è sempre true
P && false   ≡  false      // Un AND con false è sempre false
P <==> P     ≡  true       // Ogni proposizione è equivalente a sé stessa
P <=!=> P    ≡  false      // Nessuna proposizione è diversa da sé stessa
P ==> P      ≡  true       // Ogni proposizione implica sé stessa
```

### 2.12 Elementi neutri

```
P  ≡  true && P       // true è l'elemento neutro di &&
P  ≡  false || P      // false è l'elemento neutro di ||
P  ≡  true <==> P     // true è l'elemento neutro di <==>
P  ≡  false <=!=> P   // false è l'elemento neutro di <=!=>
```

> [!TIP]
> **Riepilogo pratico per la programmazione:**
>
> - `condizione && true` → si semplifica in `condizione`
> - `condizione || false` → si semplifica in `condizione`
> - `condizione && false` → si semplifica in `false` (cortocircuito)
> - `condizione || true` → si semplifica in `true` (cortocircuito)
>
> Queste semplificazioni sono sfruttate automaticamente dai compilatori (_constant folding_), ma saperle riconoscere nel codice sorgente migliora la leggibilità e la manutenibilità.

## 3. Dimostrazioni mediante catene di equivalenze

Oltre a verificare le equivalenze tramite tavole di verità, possiamo **dimostrare** nuove equivalenze applicando in sequenza le leggi già note. Questo approccio è più elegante e scalabile, perché le tavole di verità crescono esponenzialmente con il numero di variabili. [p. 10–12]

### 3.1 Esempio 1

**Dimostrare:** `(A || B) && (!A || B) ≡ B`

```
  (A || B) && (!A || B)
≡ B || (A && !A)           [distributività di || rispetto a &&, letta al contrario]
≡ B || false               [contraddizione: A && !A ≡ false]
≡ B                        [elemento neutro di ||]
```

### 3.2 Esempio 2

**Dimostrare:** `!(!(A || B) || !(A || C)) ≡ A || (B && C)`

```
  !(!(A || B) || !(A || C))
≡ !!(A || B) && !!(A || C)    [De Morgan: !(X || Y) ≡ !X && !Y]
≡ (A || B) && (A || C)        [doppia negazione, applicata due volte]
≡ A || (B && C)               [distributività di || rispetto a &&, letta al contrario]
```

> [!NOTE]
> La tecnica di dimostrazione per catena di equivalenze è esattamente quella utilizzata nei **dimostratori automatici di teoremi** e nei tool di **verifica formale** del software. Nella pratica industriale, strumenti come Z3, Alloy o i solver SAT/SMT eseguono ragionamenti analoghi su scala enormemente più grande per verificare proprietà di sicurezza del codice.

## 4. Dalle proposizioni ai predicati: funzioni proposizionali

La logica proposizionale, per quanto potente, ha un limite: non può esprimere affermazioni che si riferiscono a _insiemi_ di oggetti. Per superare questo limite, introduciamo la **logica dei predicati** (o logica del primo ordine). [p. 13]

### 4.1 Funzioni proposizionali (predicati)

Una **funzione proposizionale** è un enunciato che dipende da una o più **variabili**. A differenza di una proposizione (che è sempre vera o falsa), una funzione proposizionale **non ha un valore di verità definito** finché le variabili non vengono _istanziate_ (cioè non viene assegnato loro un valore concreto).

**Esempio:**

$$P(x) = x - 3 > 5$$

- `P` è il **predicato** (la "forma" dell'affermazione)
- `x` è la **variabile**
- `P(4) = false` poiché $4 - 3 = 1$, che non è $> 5$
- `P(10) = true` poiché $10 - 3 = 7 > 5$

### 4.2 Dai predicati alle proposizioni: i quantificatori

Per trasformare una funzione proposizionale in una proposizione (cioè in un'affermazione con valore di verità definito), utilizziamo i **quantificatori**.

---

## 5. Il quantificatore universale (∀)

Una formula **quantificata universalmente** afferma che un predicato è vero per _tutti_ gli elementi dell'universo del discorso: [p. 14]

$$\forall x \; P(x) \quad \Leftrightarrow \quad \text{"Per ogni } x, \; P(x) \text{ è vero"}$$

Si noti che `∀x P(x)` è una **proposizione** (vera o falsa), non più una funzione proposizionale: il quantificatore ha _vincolato_ la variabile libera.

### 5.1 Esempio

Siano:

- `S(x)` = _"x è uno studente universitario"_
- `G(x)` = _"x è un genio"_

Allora: [p. 15]

$$\forall x \; (S(x) \Rightarrow G(x))$$

si legge: _"Tutti gli studenti universitari sono geni."_

> [!IMPORTANT]
>
> #### Il quantificatore universale si abbina all'implicazione, non alla congiunzione
>
> - `∀x (S(x) ==> G(x))` → _"Per ogni x, **se** x è studente, **allora** x è un genio"_ ✓
> - `∀x (S(x) && G(x))` → _"Per ogni x, x è studente **e** x è un genio"_ ✗
>
> La seconda formulazione afferma che _ogni cosa nell'universo_ è contemporaneamente uno studente e un genio, il che è completamente diverso dalla frase originale. **Questo è uno degli errori di traduzione più comuni e insidiosi.** [p. 24]

### 5.2 Restrizione dell'ambito con più proprietà

Quando vogliamo restringere il quantificatore universale a elementi che soddisfano _più_ condizioni, le condizioni restrittive vengono combinate con la **congiunzione** nell'antecedente dell'implicazione. [p. 25]

**Esempio:** _"Tutti i funghi bianchi sono potenzialmente velenosi."_

Siano:

- `F(x)` = _"x è un fungo"_
- `B(x)` = _"x è bianco"_
- `V(x)` = _"x è potenzialmente velenoso"_

$$\forall x \; ((F(x) \land B(x)) \Rightarrow V(x))$$

```
∀x ((F(x) && B(x)) ==> V(x))
```

Questa è equivalente a `∀x (F(x) ==> (B(x) ==> V(x)))`, poiché:

$$A \land B \Rightarrow C \equiv A \Rightarrow (B \Rightarrow C)$$

(per la legge di esportazione/importazione, derivabile dalle equivalenze note).

### 5.3 Attenzione alla congiunzione nell'antecedente

**Esempio:** _"Tutti gli uomini e le donne sono invitati ad entrare."_ [p. 26]

Siano:

- `U(x)` = _"x è un uomo"_
- `D(x)` = _"x è una donna"_
- `I(x)` = _"x è invitato ad entrare"_

**Traduzione ERRATA:**

```
∀x ((U(x) && D(x)) ==> I(x))
```

Questa dice: _"Tutti coloro che sono contemporaneamente uomini e donne sono invitati"_ — il che non è il significato della frase originale (e peraltro l'antecedente è quasi sempre falso, rendendo l'implicazione trivialmente vera).

**Traduzione CORRETTA:**

```
∀x ((U(x) || D(x)) ==> I(x))
```

Questa dice: _"Per ogni x, **se** x è un uomo **oppure** x è una donna, **allora** x è invitato"_ — che è esattamente il significato desiderato.

> [!WARNING]
> In italiano, _"e"_ nel contesto _"uomini e donne"_ non corrisponde al connettivo `&&` ma al connettivo `||`. La traduzione dal linguaggio naturale alla logica formale richiede sempre attenzione al **significato inteso**, non alla traduzione letterale parola per parola. Questo è particolarmente rilevante nella redazione di policy di sicurezza formali.

## 6. Il quantificatore esistenziale (∃)

Una formula con **quantificatore esistenziale** afferma che esiste _almeno un_ elemento nell'universo del discorso per cui il predicato è vero: [p. 16]

$$\exists x \; P(x) \quad \Leftrightarrow \quad \text{"Esiste (almeno) un } x \text{ tale che } P(x) \text{ è vero"}$$

Anche `∃x P(x)` è una **proposizione**, non una funzione proposizionale.

### 6.1 Esempio

Siano: [p. 17]

- `P(x)` = _"x è un professore universitario"_
- `G(x)` = _"x è un genio"_

$$\exists x \; (P(x) \land G(x))$$

si legge: _"Esiste almeno un professore universitario che è un genio."_

> [!IMPORTANT]
>
> #### Il quantificatore esistenziale si abbina alla congiunzione, non all'implicazione
>
> - `∃x (P(x) && G(x))` → _"Esiste un x che è professore **e** genio"_ ✓
> - `∃x (P(x) ==> G(x))` → Formulazione problematica: sarebbe vera anche se x non è un professore (perché l'implicazione con premessa falsa è sempre vera)
>
> Mentre `∀` si abbina a `==>`, il quantificatore `∃` si abbina a `&&`.

### 6.2 Esempio con quantificatori annidati

Consideriamo l'universo dei numeri reali: [p. 18]

$$\forall x \; \exists y \; (x + y = 320)$$

Si legge: _"Per ogni x, esiste un y tale che x + y = 320."_

- **È vero per i numeri reali?** Sì, basta porre $y = 320 - x$.
- **È vero per i numeri naturali?** No! Se $x = 321$, dovremmo avere $y = -1$, che non è un numero naturale.

Questo esempio illustra come il **dominio** (universo del discorso) influenzi il valore di verità di una formula quantificata.

## 7. Traduzione dal linguaggio naturale alla logica dei predicati

La traduzione di frasi italiane in formule logiche è un'operazione non banale, perché il linguaggio naturale è ricco di sfumature e ambiguità che la logica formale non tollera. [p. 19–20]

### 7.1 Modi diversi di esprimere la congiunzione

In italiano, la congiunzione logica (`&&`) può essere espressa con molte parole diverse: [p. 20]

| Parola italiana | Connettivo logico |
| --------------- | ----------------- |
| _e_             | `&&`              |
| _ma_            | `&&`              |
| _inoltre_       | `&&`              |
| _tuttavia_      | `&&`              |
| _sebbene_       | `&&`              |
| _anche se_      | `&&`              |

Tutte queste parole, pur avendo sfumature diverse in italiano (concessione, opposizione, aggiunta), dal punto di vista _logico_ esprimono lo stesso connettivo: la congiunzione.

### 7.2 "Né… né" e "non entrambi"

Attenzione a due costrutti che sembrano simili ma sono logicamente diversi: [p. 21]

**"Né A né B"** — è vera se e solo se sia `A` sia `B` sono false:

```
!A && !B
```

Equivalente a `!(A || B)` per De Morgan.

**"A e B non sono entrambi veri"** — è vera se almeno uno dei due è falso:

```
!(A && B)
```

Equivalente a `!A || !B` per De Morgan.

**"A e B sono entrambi non veri"** — è equivalente a "né A né B":

```
!A && !B
```

> [!TIP]
> Il modo migliore per non confondersi è sempre formalizzare con calma e poi verificare con una tavola di verità a 4 righe. Un errore nella traduzione di "né… né" vs "non entrambi" in una condizione di un firewall o di un controllo di accesso potrebbe avere conseguenze gravi.

### 7.3 Modi diversi di esprimere l'implicazione

La frase `A ==> B` può essere formulata in italiano in molti modi: [p. 22]

| Formulazione italiana                    | Formula   |
| ---------------------------------------- | --------- |
| Se A allora B                            | `A ==> B` |
| Se A, B                                  | `A ==> B` |
| A implica B                              | `A ==> B` |
| A quindi B                               | `A ==> B` |
| B se A                                   | `A ==> B` |
| A è una condizione **sufficiente** per B | `A ==> B` |
| B è una condizione **necessaria** per A  | `A ==> B` |
| A **solo se** B                          | `A ==> B` |

> [!NOTE]
> L'ultima formulazione è particolarmente insidiosa: _"A solo se B"_ non significa _"solo se B allora A"_ (che sarebbe `B ==> A`), bensì _"A è vera solo nel caso in cui B sia vera"_, ovvero `A ==> B`. Questo è un punto cruciale nella redazione di specifiche formali.

### 7.4 La regola del parafrasare

Una buona norma è **parafrasare** la frase italiana in una forma più semplice prima di tradurla in logica. [p. 28]

**Esempio:** _"Nessun cane è un gatto."_

Parafrasi: _"Tutti i cani non sono gatti."_

```
∀x (C(x) ==> !G(x))
```

Che è equivalente a:

```
∀x (G(x) ==> !C(x))     // contronominale
```

E anche a:

```
!∃x (C(x) && G(x))       // "non esiste un x che sia cane e gatto"
```

Perché? Tutte queste formule si riducono a `∀x (!C(x) || !G(x))`, cioè: per ogni `x`, `x` non è un cane oppure `x` non è un gatto (o nessuno dei due).

## 8. Espressioni ricorrenti: prontuario di traduzione

La tabella seguente raccoglie le traduzioni standard più comuni, indispensabili per formalizzare rapidamente frasi del linguaggio naturale. [p. 29]

| Frase italiana            | Formula logica        |
| ------------------------- | --------------------- |
| Tutti gli A sono B        | `∀x (A(x) ==> B(x))`  |
| Nessun A è B              | `∀x (A(x) ==> !B(x))` |
| Alcuni A sono B           | `∃x (A(x) && B(x))`   |
| Alcuni A non sono B       | `∃x (A(x) && !B(x))`  |
| Tutti e soli gli A sono B | `∀x (A(x) <==> B(x))` |
| Solo gli A sono B         | `∀x (B(x) ==> A(x))`  |
| Non tutti gli A sono B    | `∃x (A(x) && !B(x))`  |
| Tutti gli A sono non B    | `∀x (A(x) ==> !B(x))` |

> [!IMPORTANT]
> Si noti che _"Nessun A è B"_ e _"Tutti gli A sono non B"_ sono **logicamente equivalenti** (stessa formula). Analogamente, _"Non tutti gli A sono B"_ e _"Alcuni A non sono B"_ sono equivalenti. Questo è un riflesso diretto delle leggi di De Morgan per i quantificatori (che vediamo nella sezione successiva).

## 9. Equivalenze nella logica dei predicati

Le leggi di equivalenza si estendono naturalmente dalla logica proposizionale alla logica dei predicati: [p. 30–31]

### 9.1 Leggi di De Morgan per i quantificatori

```
!∃x P(x)  ≡  ∀x !P(x)     // "non esiste un x con P" ≡ "per ogni x, non P"
!∀x P(x)  ≡  ∃x !P(x)     // "non per ogni x vale P" ≡ "esiste un x con non P"
∃x P(x)   ≡  !∀x !P(x)    // "esiste un x con P" ≡ "non è vero che per ogni x non vale P"
∀x P(x)   ≡  !∃x !P(x)    // "per ogni x vale P" ≡ "non esiste un x con non P"
```

Queste leggi permettono di passare da formulazioni con `∀` a formulazioni con `∃` e viceversa, esattamente come le leggi di De Morgan proposizionali permettono di passare da `&&` a `||`.

### 9.2 Scambio di quantificatori omogenei

```
∃x ∃y P(x,y)  ≡  ∃y ∃x P(x,y)
∀x ∀y P(x,y)  ≡  ∀y ∀x P(x,y)
```

Quantificatori dello stesso tipo possono essere scambiati senza alterare il significato.

### 9.3 Distributività dei quantificatori

```
∀x (P(x) && Q(x))  ≡  ∀x P(x) && ∀x Q(x)
∃x (P(x) || Q(x))  ≡  ∃x P(x) || ∃x Q(x)
```

Ma **attenzione** — le seguenti **non** sono equivalenze valide in generale: [p. 31]

```
∀x (P(x) || Q(x))  ≢  ∀x P(x) || ∀x Q(x)
∃x (P(x) && Q(x))  ≢  ∃x P(x) && ∃x Q(x)
```

#### Controesempio per `∀x (P(x) || Q(x)) ≢ ∀x P(x) || ∀x Q(x)`

Sia l'universo del discorso l'insieme degli interi, con: [p. 32]

- `P(x)` = _"x è pari"_
- `D(x)` = _"x è dispari"_

La formula `∀x (P(x) || D(x))` afferma: _"Per ogni intero x, x è pari **oppure** x è dispari"_ → **VERA** (ogni intero è l'uno o l'altro).

La formula `∀x P(x) || ∀x D(x)` afferma: _"Ogni intero è pari, **oppure** ogni intero è dispari"_ → **FALSA** (nessuna delle due è vera per tutti gli interi).

> [!WARNING]
> La non-distributività del `∀` rispetto a `||` (e del `∃` rispetto a `&&`) è una fonte comune di errori nella specifica formale di proprietà di sicurezza. Verificare che "ogni utente abbia privilegio A oppure privilegio B" **non** equivale a dire "tutti hanno A oppure tutti hanno B".

## 10. Esercizi di traduzione

### 10.1 Esercizio: "Nessun giocatore ama perdere" [p. 33–34]

Siano:

- `G(x)` = _"x è un giocatore"_
- `P(x)` = _"x ama perdere"_

**Traduzione:**

```
∀x (G(x) ==> !P(x))
```

Che è equivalente a:

```
∀x (P(x) ==> !G(x))       // contronominale
!∃x (G(x) && P(x))         // De Morgan per quantificatori
```

Tutte e tre le forme si riducono a `∀x (!G(x) || !P(x))`: per ogni `x`, `x` non è un giocatore oppure `x` non ama perdere.

### 10.2 Esercizio: "Ogni intero positivo e ogni intero negativo hanno il quadrato positivo" [p. 35]

Siano:

- `P(x)` = _"x è un intero positivo"_
- `N(x)` = _"x è un intero negativo"_
- `Q(x)` = _"il quadrato di x è positivo"_

**Traduzione corretta:**

```
∀x ((P(x) || N(x)) ==> Q(x))
```

**Traduzione ERRATA:**

```
∀x ((P(x) && N(x)) ==> Q(x))    // SBAGLIATO: "x è positivo E negativo" è sempre falso
```

**Formulazione alternativa equivalente:**

```
∀x (P(x) ==> Q(x)) && ∀x (N(x) ==> Q(x))
```

Questa è corretta perché sfrutta la distributività di `∀` rispetto a `&&`.

### 10.3 Esercizio: "Esiste un intero che non è né positivo né negativo" [p. 36]

Siano:

- `I(x)` = _"x è un intero"_
- `P(x)` = _"x è positivo"_
- `N(x)` = _"x è negativo"_

```
∃x (I(x) && !P(x) && !N(x))
```

La soluzione è l'intero $0$, che per convenzione non è né positivo né negativo.

## 11. Booleani ed espressioni logiche in JML

Java Modeling Language (**JML**) è un linguaggio di specifica formale per programmi Java. In questa sezione colleghiamo la logica dei predicati alla sintassi concreta che utilizzeremo nel laboratorio. [p. 37]

### 11.1 Costanti e operatori

| Categoria              | Sintassi JML                              |
| ---------------------- | ----------------------------------------- |
| Costanti booleane      | `true`, `false`                           |
| Operatori booleani     | `!`, `&&`, `\|\|`, `==>`, `<==>`, `<=!=>` |
| Operatori di confronto | `==`, `!=`, `>`, `<`, `>=`, `<=`          |

Gli operatori di confronto sono di fatto **predicati binari atomici** scritti in notazione **infissa**. Ad esempio:

- `x == y` è il predicato `Uguale(x, y)` scritto come `U(x, y)` in notazione prefissa
- `x != y` è il predicato `Diverso(x, y)` scritto come `D(x, y)`
- `x == 2` è `true` se e solo se la variabile `x` è uguale a 2

### 11.2 Predicati composti

I predicati atomici possono essere combinati con i connettivi logici per formare **predicati composti**, esattamente come le proposizioni atomiche formano proposizioni composte:

```java
// Esempio con x e i interi, e A array di interi:
(x == 2 || A[i] < 9)
```

## 12. Applicazione pratica: specifiche formali su array

Gli esempi che seguono mostrano come le formule con quantificatori vengano utilizzate per specificare proprietà di **array** — una struttura dati fondamentale nella programmazione. [p. 38–42]

### 12.1 Esempio: "L'array A non contiene elementi maggiori di 0"

Sia `A` un array di interi con `A.length` elementi. [p. 38]

**Formulazione con ∀ (quantificatore universale):**

```
∀i ((i >= 0 && i < A.length) ==> !(A[i] > 0))
```

Si legge: _"Per ogni indice i valido, l'elemento A[i] non è maggiore di 0."_

**Formulazione con ∃ negato:**

```
!∃i ((i >= 0 && i < A.length) && (A[i] > 0))
```

Si legge: _"Non esiste un indice i valido tale che A[i] sia maggiore di 0."_

#### Dimostrazione dell'equivalenza

```
  !∃i ((i >= 0 && i < A.length) && (A[i] > 0))
≡ ∀i !((i >= 0 && i < A.length) && (A[i] > 0))      [De Morgan per ∃]
≡ ∀i (!(i >= 0 && i < A.length) || !(A[i] > 0))      [De Morgan proposizionale]
≡ ∀i ((i >= 0 && i < A.length) ==> !(A[i] > 0))      [P ==> Q ≡ !P || Q]
```

> [!NOTE]
> Il pattern `∀i ((i >= 0 && i < A.length) ==> ...)` è la forma standard per esprimere proprietà universali su tutti gli elementi di un array. L'antecedente `i >= 0 && i < A.length` è la **guardia** che restringe `i` agli indici validi, proteggendo da accessi fuori limite (_buffer overflow_) — una delle vulnerabilità più sfruttate nella storia della sicurezza informatica.

### 12.2 Esempio: "Se la dimensione è positiva, l'array contiene tutti elementi negativi"

Sia `dim` una variabile intera. [p. 39]

```
(dim >= 0) ==> ∀i ((i >= 0 && i < A.length) ==> (A[i] < 0))
```

Questa specifica è un'**implicazione** il cui antecedente è una condizione sulla variabile `dim` e il cui conseguente è una proprietà universale sull'array. Se `dim < 0`, l'implicazione è trivialmente vera; se `dim >= 0`, allora ogni elemento dell'array deve essere negativo.

### 12.3 Esercizio 1: "Gli elementi in posizione pari di A sono diversi da 9" [p. 40–41]

```
∀i ((i >= 0 && i < A.length && i % 2 == 0) ==> (A[i] != 9))
```

Si noti come la condizione `i % 2 == 0` sia stata aggiunta nell'antecedente dell'implicazione, insieme alla guardia sugli indici validi. Questo restringe ulteriormente l'ambito del quantificatore ai soli indici pari.

### 12.4 Esercizio 2: "Gli elementi di A sono in ordine crescente" [p. 42]

```
∀i ((i >= 0 && i < A.length - 1) ==> (A[i] <= A[i+1]))
```

Si noti che il limite superiore è `A.length - 1` (e non `A.length`), poiché confrontiamo ogni elemento con il **successivo** `A[i+1]`: l'ultimo elemento non ha un successore. Questo dettaglio è fondamentale per evitare un accesso fuori dai limiti dell'array.

> [!WARNING]
> Errori di tipo _off-by-one_ (sbagliare il limite di un indice di ±1) sono tra le cause più frequenti di **buffer overflow** nei programmi C/C++ e, più in generale, di comportamenti anomali in qualsiasi linguaggio. La specifica formale ci obbliga a ragionare esplicitamente sui limiti, rendendo evidenti errori che nel codice informale passerebbero inosservati.

## Riepilogo dei concetti chiave

| Concetto                         | Descrizione                                                |
| -------------------------------- | ---------------------------------------------------------- |
| **Equivalenza** (`≡`)            | Due formule con la stessa tavola di verità                 |
| **Idempotenza**                  | `P \|\| P ≡ P`, `P && P ≡ P`                               |
| **Commutatività**                | Vale per `\|\|`, `&&`, `<==>`, `<=!=>` — **non** per `==>` |
| **Associatività**                | Vale per `\|\|`, `&&`, `<==>`, `<=!=>`                     |
| **De Morgan**                    | `!(P \|\| Q) ≡ !P && !Q` e `!(P && Q) ≡ !P \|\| !Q`        |
| **Contronominale**               | `P ==> Q ≡ !Q ==> !P`                                      |
| **Funzione proposizionale**      | Enunciato con variabili libere (predicato)                 |
| **Quantificatore ∀**             | "Per ogni x" — si abbina a `==>`                           |
| **Quantificatore ∃**             | "Esiste un x" — si abbina a `&&`                           |
| **De Morgan per quantificatori** | `!∀x P ≡ ∃x !P` e `!∃x P ≡ ∀x !P`                          |
| **Specifica su array**           | `∀i ((guardia) ==> (proprietà))`                           |
